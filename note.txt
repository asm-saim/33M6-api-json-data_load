API(Application Programming Interface)
An API (Application Programming Interface) is a set of rules that allows different software applications to communicate with each other.

Think of it like a waiter in a restaurant:
You (the user) place an order.
The waiter (the API) takes your order to the kitchen.
The kitchen (the server/application) prepares the food.
The waiter brings the food back to you.

Summary: An API is a way for software systems to talk to each other and exchange data or functionality.

Common uses:
1. Fetching weather data in an app
2. Logging in with Google/Facebook
3. Processing payments via Stripe
4. Getting tweets from Twitter
---------------------------------------------------------------------
//Types of API Requests (HTTP Methods)

METHOD      FULL NAME        WHAT IT DOES                EXAMPLE              
GET         Retrieve         Gets data from server       Get user list, posts 
POST        Create           Sends new data to server    Create a new user    
PUT         Replace/Update   Replaces full data          Update entire user info  
PATCH       Partial Update   Updates part of data        Update only name/email  
DELETE      Delete           Removes data                Delete a user                    |

//HTTP vs HTTPS
Both are protocols for transferring data between a client (browser/app) and a server — the difference is security.

Feature                           HTTP                   HTTPS 
Full name              HyperText Transfer Protocol   HTTP Secure
Encryption             ❌ None                      ✅ SSL/TLS encrypted
Data safety            Exposed in plain text         Encrypted, unreadable to outsiders
Port                   80                            443
URL starts wit         http://                      https://
Use case               Old/internal sites            All modern websites & APIs

/// JSON: JavaScript Object Notation.

*It is a lightweight format used to store and exchange data between applications, especially when working with APIs.

**JSON Structure: JSON stores data as key-value pairs.  key are in "double quotes", 
but not number , true/false 

**Differences
JavaScript Object	    JSON
Keys can be unquoted	Keys must be in double quotes
Can contain functions	Cannot contain functions
Used inside JavaScript	Used for data exchange

//Why JSON Is Important
APIs send and receive data in JSON format.
Easy for humans to read.
Easy for machines to parse.
Supported by almost every programming language.

//Fetch:
fetch() is a JavaScript function used to send requests to a server (API) and get data back.

// fetch() does not return the data immediately.
// It returns a Promise because the request takes time.

fetch('https://jsonplaceholder.typicode.com/todos/1')
    .then(res => res.json()) // .json() -> parse the jason data into object
    .then(json => console.log(json))

// fetch() is a built-in JavaScript function used to make HTTP requests to servers or APIs. It returns a Promise that resolves to a Response object, which can be converted into JavaScript data using methods such as response.json().

async/await:
async creates a Promise-based function, and await waits for a Promise to resolve before continuing execution inside that function.

//
Why Do We Need Them?

Fetching data takes time.

Imagine:

console.log("Start");
fetch(url);
console.log("End");

Output:

Start
End

*JavaScript doesn't wait for the API response.

*That's because fetch() is asynchronous.


async: async marks a function as asynchronous and makes it automatically return a Promise.

await: await pauses the execution of an async function until a Promise resolves and returns its resolved value.
